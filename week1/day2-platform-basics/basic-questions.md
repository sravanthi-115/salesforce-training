# 📘 Day 2 Evaluation Questions

---

## 1️⃣ What is an App in Salesforce?

An App in Salesforce is a collection of related tabs, objects, and features designed for a specific business purpose.

It helps users access required tools and data from one place.

### Examples
- Sales App
- Service App
- College Admission App

---

## 2️⃣ What is an Object?

An Object in Salesforce is like a database table used to store information.

### Examples
- **Account Object** → Stores company details
- **Contact Object** → Stores customer/person details
- **Opportunity Object** → Stores sales deal information

Objects contain:
- Records
- Fields

---

## 3️⃣ Difference Between App and Object

| App | Object |
|------|------|
| Collection of features and tabs | Stores data |
| Used for navigation and workflow | Used for maintaining records |
| Contains multiple objects | Represents a single data entity |
| Example: Sales App | Example: Account Object |

---

## 4️⃣ What is Multi-Tenant Architecture?

Multi-tenant architecture means multiple companies use the same Salesforce infrastructure and servers while keeping their data secure and separate.

### Benefits
- Cost effective
- Scalable
- Easy maintenance
- Shared resources

---

## 5️⃣ When Should We Use Configuration Instead of Code?

Configuration should be used when requirements can be achieved using Salesforce built-in tools without programming.

### Examples
- Creating custom fields
- Building workflows and flows

### Advantages
- Faster development
- Easier maintenance
- No coding knowledge required

---

## 6️⃣ How Does Salesforce Allow Developers to Extend Functionality?

Salesforce allows developers to extend functionality using:

- Apex programming language
- Lightning Components
- APIs
- Custom Objects
- Automation tools

These tools help developers build custom business logic, integrations, and user interfaces according to business needs.
