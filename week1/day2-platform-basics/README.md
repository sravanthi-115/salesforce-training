# Day 2 - Salesforce Platform Basics

## What is Salesforce Platform?

Salesforce Platform is a cloud-based platform that helps businesses build applications, manage customer data, automate workflows, and extend functionality using both configuration tools and coding.

It provides:

- CRM features
- Application development tools
- Automation
- Cloud storage
- Security and scalability

---

# Salesforce Platform Concepts

## App

An App in Salesforce is a collection of tabs, objects, and functionalities designed for a specific business purpose.

Example:

- Sales App
- Service App
- College Admission App

Apps help users access related features from one place.

---

## Object

An Object is a database table used to store information in Salesforce.

Examples:

- Account Object
- Contact Object
- Opportunity Object

Custom objects can also be created based on business requirements.

---

## Tab

A Tab is the user interface used to access objects, records, dashboards, reports, or applications.

Tabs help users navigate inside Salesforce.

Example:

- Accounts Tab
- Contacts Tab
- Opportunities Tab

---

# Connecting Day 1 and Day 2

CRM concepts fit into Salesforce Platform using Objects and Apps.

Example:

- Account → Stored as Account Object
- Contact → Stored as Contact Object
- Opportunity → Stored as Opportunity Object

These objects are grouped together inside apps like the Sales App.

Users interact with records through tabs and user interfaces.

---

# Configuration vs Coding

## Configuration (No Code)

Configuration means building functionality using clicks, drag-and-drop tools, and setup options without writing code.

Examples:

1. Creating custom fields
2. Building flows and automation

Advantages:

- Faster development
- Easier maintenance
- No programming required

---

## Coding (Apex)

Coding is used when advanced business logic or custom functionality is required.

Salesforce mainly uses Apex programming language.

Examples:

1. Complex validation logic
2. Custom API integrations

Advantages:

- More flexibility
- Advanced customization
- Complex automation support

---

# Multi-Tenant Architecture

Salesforce uses multi-tenant architecture.

This means:

- Multiple companies use the same Salesforce infrastructure
- Each company’s data remains secure and separate
- Resources are shared efficiently

This makes Salesforce scalable and cost-effective.
